Дорожная карта документации по БД Oracle

Набор документации разработан с учетом конкретных путей доступа, чтобы пользователи могли максимально эффективно находить необходимую им информацию.

Набор документации разделен на три уровня или группы: базовый, промежуточный и продвинутый. Пользователи начинают с руководств в базовой группе, переходят к руководствам в промежуточной группе (серия 2 дня +) и, наконец, к расширенным руководствам, которые включают остальную документацию.

Документацию по поддерживаемым выпускам БД Oracle можно найти по следующему адресу

Документация по БД Oracle: Базовая группа

Технические пользователи, новички в Oracle Database, начинают с прочтения одного или нескольких руководств в базовой группе от корки до корки. Каждое руководство в этой группе рассчитано на прочтение в течение двух дней.

В дополнение к этому руководству в базовую группу входят руководства, приведенные в следующей таблице.

Таблица 1-1 Базовая группа

Руководство Описание
Oracle Database 2 Day Developer’s Guide В этом кратком руководстве по выполнению задач объясняется, как использовать основные функции Oracle Database с помощью SQL и PL/SQL.

Руководства, входящие в базовую группу, тесно связаны между собой, что отражается в количестве перекрестных ссылок. Например, Oracle Database Concepts часто отправляет пользователей на 2-дневное руководство, чтобы узнать, как выполнить задачу на основе концепции. В 2-дневных руководствах часто упоминаются концепции баз данных Oracle для получения концептуальной информации о задаче.

Документация по БД Oracle: Промежуточная группа

Следующим шагом по сравнению с базовой группой является промежуточная группа.

Руководства в промежуточной группе имеют префикс со словом 2 дня +, потому что они расширяют и предполагают информацию, содержащуюся в 2-дневных руководствах. Руководства на 2 дня + охватывают темы более глубоко, чем это возможно в базовых руководствах, или охватывают темы, представляющие особый интерес. Руководства предназначены для разных аудиторий:

Администраторы БД

Oracle Database 2 Day + Performance Tuning Guide – это краткое руководство, в котором описывается, как выполнять повседневные задачи по настройке производительности базы данных с использованием функций, предоставляемых Oracle Diagnostics Pack, Oracle Tuning Pack и Oracle Enterprise Manager Cloud Control (облачное управление).

Разработчики БД

Oracle Database 2 Day + Java Developer’s Guide поможет вам разобраться во всех продуктах Java, используемых для создания Java-приложения. В руководстве объясняется, как использовать Oracle JDBC Thin driver, Универсальный пул подключений (UCP) и Java в базе данных (JVM) в примере веб-приложения.

Документация по БД Oracle: Продвинутая группа

Руководства для продвинутых групп предназначены для опытных пользователей, которым требуется более подробная информация по конкретной теме, чем может быть предоставлена в руководствах 2 дня +.

В следующей таблице перечислены основные справочные руководства в группе advanced.

Таблица 1-2 Основные справочные руководства

Руководство Описание
Oracle Database SQL Language Reference Содержит полное описание языка структурированных запросов (SQL), используемого для управления информацией в базе данных Oracle.
Oracle Database Reference Описывает параметры инициализации базы данных, представления словаря данных, представления динамической производительности, события ожидания и фоновые процессы.
Oracle Database PL/SQL Packages and Types Reference Описывает пакеты PL/SQL, поставляемые с сервером баз данных Oracle. Вы можете использовать поставляемые пакеты при создании своих приложений или для поиска идей при создании собственных хранимых процедур.

Расширенных руководств слишком много, чтобы перечислять их в этом разделе. В следующей таблице перечислены руководства, которые использует большинство опытных администраторов баз данных Oracle.

Таблица 1-3 Продвинутая группа для Администраторов БД

Руководство Описание
Oracle Database Administrator’s Guide Объясняет, как выполнять такие задачи, как создание и настройка баз данных, обслуживание и мониторинг баз данных, создание объектов схемы, планирование заданий и диагностика проблем.
Oracle Database Security Guide Описывает, как настроить безопасность для базы данных Oracle с помощью функций базы данных по умолчанию.
Oracle Database Performance Tuning Guide Описывает, как использовать инструменты Oracle Database tools для оптимизации производительности базы данных. В этом руководстве также описываются рекомендации по повышению производительности при создании базы данных и содержится справочная информация, связанная с производительностью.
Oracle Database SQL Tuning Guide Describes SQL processing, the optimizer, execution plans, SQL operators, optimizer statistics, application tracing, and SQL advisors.
Oracle Database Backup and Recovery User’s Guide Объясняет, как создавать резервные копии, восстанавливать базы данных Oracle, выполнять обслуживание резервных копий файлов базы данных и передавать данные между системами хранения.
Oracle Real Application Clusters Administration and Deployment Guide Объясняет, как установить, настроить, управлять и устранять неполадки в базе данных Oracle RAC.

В следующей таблице перечислены руководства, которые использует большинство опытных разработчиков Oracle.

Таблица 1-4 Продвинутая группа для Разработчиков

Руководство Описание
Oracle Database Development Guide Объясняет, как разрабатывать приложения или преобразовывать существующие приложения для запуска в среде базы данных Oracle. В руководстве объясняются основы проектирования приложений и описываются основные концепции разработки на SQL и PL/SQL.
Oracle Database PL/SQL Language Reference Описывает все аспекты языка PL/SQL, включая типы данных, управляющие инструкции, коллекции, триггеры, пакеты и обработку ошибок.
Oracle Database Java Developer’s Guide Описывает, как разрабатывать, загружать и запускать Java-приложения в базе данных Oracle.
Oracle Database SecureFiles and Large Objects Developer’s Guide Объясняет, как разрабатывать новые приложения с использованием больших объектов (LOBs), LOBs SecureFiles и файловой системы базы данных (DBFS).

Другие расширенные руководства, требуемые конкретным пользователем, зависят от сферы ответственности этого пользователя.
