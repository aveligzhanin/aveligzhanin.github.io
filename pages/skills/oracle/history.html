Краткая история БД Oracle

Текущая версия БД Oracle является результатом более чем 40-летнего инновационного развития.

Основные моменты в развитии Oracle Database включают следующее:

Основание корпорации Oracle

В 1977 году Ларри Эллисон, Боб Майнер и Эд Оутс основали консалтинговую компанию Software Development Laboratories, которая стала Relational Software, Inc. (RSI). В 1983 году RSI стала корпорацией Oracle Systems, а затем и Oracle Corporation.

Первая коммерчески доступная СУБД

В 1979 году RSI представила Oracle V2 (версия 2) в качестве первой коммерчески доступной РСУБД на базе SQL, что стало знаковым событием в истории реляционных баз данных.

Портируемая версия БД Oracle

Oracle версии 3, выпущенная в 1983 году, была первой реляционной базой данных, работающей на мэйнфреймах, миникомпьютерах и персональных компьютерах. База данных была написана на C, что позволило перенести ее на несколько платформ.

Улучшения в управлении параллелизмом, распределении данных и масштабируемости

В версии 4 введена мультиверсионная согласованность чтения. Версия 5, выпущенная в 1985 году, поддерживала клиент-серверные вычисления и распределенные системы баз данных. Версия 6 внесла улучшения в дисковый ввод-вывод, блокировку строк, масштабируемость, а также резервное копирование и восстановление. Кроме того, в версии 6 была представлена первая версия языка PL/SQL, проприетарного процедурного расширения для SQL.

Модули хранимых процедур PL/SQL

Oracle 7, выпущенная в 1992 году, представила хранимые процедуры и триггеры PL/SQL.

Объекты и партиционирование

Oracle 8 была выпущена в 1997 году как объектно-реляционная база данных, поддерживающая множество новых типов данных. Кроме того, Oracle 8 поддерживала секционирование больших таблиц.

Интернет-вычисления

База данных Oracle 8i, выпущенная в 1999 году, обеспечивала встроенную поддержку интернет-протоколов и серверную поддержку Java. Oracle 8i был разработан для интернет-вычислений, что позволяет развертывать базу данных в многоуровневой среде.

Кластеры реальных приложений Oracle (Oracle RAC)

База данных Oracle9i представила Oracle RAC в 2001 году, что позволило нескольким экземплярам одновременно получать доступ к одной базе данных. Кроме того, Oracle XML Database (Oracle XML DB) представила возможность хранения и запроса XML.

Сеточные вычисления

Oracle Database 10g представила grid-вычисления в 2003 году. Этот выпуск позволил организациям виртуализировать вычислительные ресурсы путем создания сетевой инфраструктуры на основе недорогих товарных серверов. Ключевой целью было сделать базу данных самоуправляемой и самонастраивающейся. Oracle Automatic Storage Management (Oracle ASM) помогла достичь этой цели за счет виртуализации и упрощения управления хранилищем баз данных.

Управляемость, диагностируемость и доступность

Oracle Database 11g, выпущенная в 2007 году, представила множество новых функций, которые позволили администраторам и разработчикам быстро адаптироваться к меняющимся бизнес-требованиям. Ключом к адаптации является упрощение информационной инфраструктуры за счет консолидации информации и использования автоматизации везде, где это возможно.

Подключение к облаку

Oracle Database 12c, выпущенная в 2013 году, была разработана для облака, отличаясь новой архитектурой с несколькими пользователями, хранилищем столбцов в памяти (IM column store) и поддержкой документов JSON. Oracle Database 12c помогла администраторам баз данных более эффективно использовать свои ИТ-ресурсы, продолжая при этом снижать затраты и повышать уровень обслуживания конечных пользователей.

Интеграция и производительность памяти

Oracle Database 18c упростила интеграцию со службами каталогов, такими как Microsoft Active Directory. В нем также была введена функциональность для использования памяти для столбчатых моделей данных и высокоскоростного доступа к строкам.

Улучшенная стабильность

Oracle Database 19c была версией семейства продуктов Oracle Database 12c (выпуск 12.2) с длительной поддержкой. Основным направлением этого выпуска была стабильность. Oracle Database 19c также внесла несколько небольших, но существенных улучшений в такие функции, как JSON и Active Data Guard.

Усовершенствование опыта разработки

Oracle Database 21c улучшает работу разработчиков с помощью таких функций, как таблицы Oracle Blockchain и собственные типы данных JSON. Усовершенствования автоматического сохранения в памяти делают хранилище столбцов обмена мгновенными сообщениями в значительной степени самоуправляемым.
